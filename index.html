Three-stage design,
kept understandable.

BootloaderFox is meant to be ambitious, but not messy. Each layer has a clear purpose so you can grow the project without turning it into spaghetti.

Stage 1

Minimal loader

Assembly/C boot path that initializes the machine, prepares memory, and hands off cleanly into the main environment.

Stage 2

Core boot engine

Filesystem access, config parsing, Linux kernel loading, and module handling live here so the project stays structured.

Stage 3

UI layer

Framebuffer graphics, keyboard and optional mouse support, plus light animation hooks for a better on-screen experience.

Why it feels different

It isn’t only about colors. The project is shaped around usability, visual clarity, and a modular future instead of carrying every old assumption forever.
